# 卡组代码接入战斗系统说明

## 放置位置

将以下 5 个文件放入 Unity 项目：

```text
Assets/Scripts/Cards/
  ElementType.cs
  CardType.cs
  DeckPreset.cs
  CardData.cs
  CardDeckLibrary.cs
```

命名空间为：

```csharp
using ElementalSpire.Cards;
```

## 卡牌数据职责

`CardDeckLibrary` 只负责提供卡牌数据，不负责战斗结算。

战斗逻辑建议由以下模块处理：

```text
BattleManager
  控制战斗开始、回合开始、回合结束

DeckManager
  管理 drawPile / handCards / discardPile / exhaustPile

CardEffectResolver
  根据 cardId 执行具体卡牌效果

ElementReactionResolver
  处理元素附着和元素反应

BattleUI
  把 CardData 显示成按钮和文字
```

## 读取卡牌数据

获取全部卡牌：

```csharp
using ElementalSpire.Cards;

IReadOnlyList<CardData> allCards = CardDeckLibrary.GetAllCards();
```

按牌组获取卡牌：

```csharp
List<CardData> fireDeck = CardDeckLibrary
    .GetCardsByDeckPreset(DeckPreset.Fire)
    .ToList();
```

按 ID 获取单张卡牌：

```csharp
CardData cardData = CardDeckLibrary.GetCardById("fire_double_strike");
```

using System.Collections.Generic;
using System.Linq;

public class BattleManager
{
    private List<CardData> drawPile = new List<CardData>();
    private List<CardData> handCards = new List<CardData>();
    private List<CardData> discardPile = new List<CardData>();
    private List<CardData> exhaustPile = new List<CardData>();

    public void StartBattle()
    {
        drawPile = CardDeckLibrary
            .GetCardsByDeckPreset(DeckPreset.Fire)
            .ToList();

        ShuffleDrawPile();
        DrawCards(5);
    }

    public void DrawCards(int drawCount)
    {
        for (int i = 0; i < drawCount; i++)
        {
            if (drawPile.Count == 0)
            {
                ReshuffleDiscardPile();
            }

            if (drawPile.Count == 0)
            {
                return;
            }

            CardData cardData = drawPile[0];
            drawPile.RemoveAt(0);
            handCards.Add(cardData);
        }
    }

    private void ReshuffleDiscardPile()
    {
        drawPile.AddRange(discardPile);

        ShuffleDrawPile();

## 元素附着规则

建议由 `ElementReactionResolver` 统一处理，不要散落在每张卡牌里。

当前规则：

- 使用元素攻击敌人时，如果敌人当前没有元素附着，则附着本次攻击元素。
- 如果敌人已经附着相同元素，本次攻击后仍保持当前元素附着。
- 如果敌人已经附着其他元素，本次攻击触发一次元素反应。
- 普通元素反应结束后，当前元素附着消失；本次攻击元素不会继续留在敌人身上。
- 如果直到敌人回合开始时普通元素仍未反应，该元素附着消失。
- 只要敌人处于中毒状态，敌人就持续拥有毒元素附着。
- 火或水攻击命中中毒敌人时，照常与毒元素触发反应；只要中毒层数仍大于 0，反应后毒元素附着不会消失。
- 施加中毒会让敌人的有效元素附着变为毒；中毒层数归零后，毒元素附着才消失。

推荐提供一个有效附着方法：

```csharp
public ElementType GetEffectiveAttachment(EnemyState enemyState)
{
    if (enemyState.poison > 0)
    {
        return ElementType.Poison;
    }

    return enemyState.temporaryAttachment;
}
```

## 接入建议

第一步只接以下流程：

```text
StartBattle()
DrawCards(5)
PlayCard(cardData)
ResolveCardEffect(cardData)
EndPlayerTurn()
```

先让卡牌能被抽到、显示、打出、进入弃牌堆或消耗区，再逐步补齐所有卡牌效果。
