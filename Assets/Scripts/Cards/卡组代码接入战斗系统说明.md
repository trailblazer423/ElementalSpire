## 当前规则口径

- 初始牌组固定为 5 张无色打击 + 5 张无色防御。
- 开局选择 2 种不同元素作为双主元素。
- 开局选牌从双主元素的普通/稀有牌中出现，珍贵牌不出现。
- 每场战斗结束后进行 2 次奖励：元素 A 一次三选一，元素 B 一次三选一，均可跳过。
- 第三元素可以通过事件、商店或无色桥接牌获得；获得后正常参与元素附着和元素反应，只是不作为战斗后常规奖励的保底来源。
- 水源不会每回合默认回复，主要由水元素牌和水脉共鸣提供。
- 毒元素附着和中毒层数独立：毒攻击或上毒都会附着一次毒元素；毒附着被反应或进入下一回合后清除，中毒层数继续保留并正常结算。

## 升级联动方式

升级状态放在玩家拥有的牌实例上，不要直接修改 CardData。CardData 是全局静态表；CardInstance 才代表玩家牌组里的一张具体牌。

```csharp
[System.Serializable]
public sealed class PlayerDeck
{
    public List<CardInstance> cards = new List<CardInstance>();

    public void UpgradeCard(int index)
    {
        if (index < 0 || index >= cards.Count)
        {
            return;
        }

        CardData data = cards[index].GetCardData();
        if (!data.hasUpgrade)
        {
            return;
        }

        cards[index].isUpgraded = true;
    }
}
```

UI 或战斗结算读取费用时：

```csharp
CardInstance cardInstance = cardInstances[0];
CardData data = cardInstance.GetCardData();

int energyCost = data.GetEnergyCost(cardInstance.isUpgraded);
int waterCost = data.GetWaterCost(cardInstance.isUpgraded);
string baseText = data.description;
string upgradeText = data.upgradeDescription;
```

如果队友的牌组系统已经有自己的牌实例类，只需要保留同等字段即可：

```csharp
public string cardId;
public bool isUpgraded;
```

战斗效果分发时根据 cardId 和 isUpgraded 分支，例如：

```csharp
void ResolveCardEffect(CardInstance cardInstance)
{
    CardData data = cardInstance.GetCardData();

    switch (data.cardId)
    {
        case "starter_strike":
            DealDamage(cardInstance.isUpgraded ? 9 : 6);
            break;

        case "starter_defend":
            GainBlock(cardInstance.isUpgraded ? 8 : 5);
            break;
    }
}
```

## CardData 字段

```csharp
cardData.cardId              // 内部逻辑 ID，例如 fire_double_strike
cardData.cardNumber          // 元素内编号；基础牌为 0
cardData.elementType         // Fire / Poison / Water / Colorless
cardData.cardName            // UI 显示名称
cardData.rarity              // 基础 / 普通 / 稀有 / 珍贵
cardData.cardType            // Attack / Skill / Defense / Power / Trick
cardData.cost                // 基础能量费用
cardData.waterCost           // 基础水源费用
cardData.description         // 基础效果文本
cardData.upgradeDescription  // 升级后效果或升级变化说明
cardData.upgradedCost        // 升级后能量费用；-1 表示不覆盖基础费用
cardData.upgradedWaterCost   // 升级后水源费用；-1 表示不覆盖基础水源费用
cardData.hasUpgrade          // 是否有升级设计
cardData.exhaust             // 是否消耗
cardData.chooseElement       // 是否需要选择火、毒、水之一打出
cardData.starter             // 是否为初始基础牌
cardData.startEligible       // 是否进入开局双元素选牌池
cardData.rewardStage         // 1-10关 / 4-10关 / 8-10关；空值代表特殊来源
cardData.keywords            // 关键词，如 打击、燃血、抽弃、蓄水
```

## 元素反应接入要点

CardDeckLibrary 只提供卡牌数据和牌池筛选，不负责战斗规则结算。

毒相关实现不要再用“中毒层数 > 0”推导毒附着。推荐敌人状态至少拆成：

```csharp
public int poisonStacks;
public ElementType elementAttachment;
public bool deepPoison;
```

毒牌处理规则：

```csharp
// 毒攻击：先按毒元素攻击尝试附着或反应，再造成攻击伤害。
DealAttackDamage(damage, hitCount, ElementType.Poison);

// 上毒：也要尝试附着或反应一次毒元素，然后增加中毒层数。
ResolveElementAttachment(ElementType.Poison);
ApplyPoisonStacks(amount);
```

## 奖励稀有度曲线

| 阶段 | 普通 | 稀有 | 珍贵 |
| --- | ---: | ---: | ---: |
| 开局选牌 | 80% | 20% | 0% |
| 第 1-3 关战斗奖励 | 75% | 25% | 0% |
| 第 4-7 关战斗奖励 | 55% | 35% | 10% |
| 第 8-10 关战斗奖励 | 35% | 40% | 25% |

具体抽样权重由 RewardResolver 实现；本文件夹只提供可被抽样的候选池。
