# 卡组代码接入战斗系统说明

## 文件位置

本文件夹提供卡牌数据与牌池筛选工具：

```text
Assets/Scripts/Cards/
  ElementType.cs
  CardType.cs
  DeckPreset.cs
  CardData.cs
  CardDeckLibrary.cs
```

命名空间：

```csharp
using ElementalSpire.Cards;
```

## 当前规则口径

- 初始牌组固定为 5 张无色打击 + 5 张无色防御。
- 开局选择 2 种不同元素作为双主元素。
- 开局选牌从双主元素的普通/稀有牌中出现，珍贵牌不出现。
- 每场战斗结束后进行 2 次奖励：元素 A 一次三选一，元素 B 一次三选一，均可跳过。
- 第三元素可以通过事件、商店或无色桥接牌获得；获得后正常参与元素附着和元素反应，只是不作为战斗后常规奖励的保底来源。
- 水源不会每回合默认回复，主要由水元素牌和水脉共鸣提供。
- 毒元素附着和中毒层数独立：毒攻击或上毒都会附着一次毒元素；毒附着被反应或进入下一回合后清除，中毒层数继续保留并正常结算。

## 读取卡牌数据

```csharp
IReadOnlyList<CardData> allCards = CardDeckLibrary.GetAllCards();
CardData cardData = CardDeckLibrary.GetCardById("fire_double_strike");
```

初始牌组：

```csharp
List<CardData> starterDeck = CardDeckLibrary.GetStarterDeck().ToList();
```

开局双元素选牌池：

```csharp
List<CardData> initialPool = CardDeckLibrary
    .GetInitialDraftPool(ElementType.Fire, ElementType.Poison)
    .ToList();
```

战斗后奖励池：

```csharp
int floor = 6;
List<CardData> fireReward = CardDeckLibrary
    .GetBattleRewardPool(ElementType.Fire, floor)
    .ToList();

List<CardData> poisonReward = CardDeckLibrary
    .GetBattleRewardPool(ElementType.Poison, floor)
    .ToList();
```

商店珍贵牌：

```csharp
List<CardData> shopCards = CardDeckLibrary.GetPreciousCards().ToList();
```

## CardData 字段

```csharp
cardData.cardId         // 内部逻辑 ID，例如 fire_double_strike
cardData.cardNumber     // 元素内编号；基础牌为 0
cardData.elementType    // Fire / Poison / Water / Colorless
cardData.cardName       // UI 显示名称
cardData.rarity         // 基础 / 普通 / 稀有 / 珍贵
cardData.cardType       // Attack / Skill / Defense / Power / Trick
cardData.cost           // 能量费用
cardData.waterCost      // 水源费用
cardData.description    // 卡牌效果文本
cardData.exhaust        // 是否消耗
cardData.chooseElement  // 是否需要选择火、毒、水之一打出
cardData.starter        // 是否为初始基础牌
cardData.startEligible  // 是否进入开局双元素选牌池
cardData.rewardStage    // 1-10关 / 4-10关 / 8-10关；空值代表特殊来源
cardData.keywords       // 关键词，如 打击、燃血、抽弃、蓄水
```

## 元素反应接入要点

战斗系统建议把元素附着、持续状态和卡牌效果拆开：

```text
CardDeckLibrary        只提供卡牌数据和牌池筛选
CardEffectResolver     根据 cardId 执行具体效果
ElementReactionResolver 处理元素附着和元素反应
DeckManager            管理抽牌堆、手牌、弃牌堆、消耗区
RewardResolver         根据双主元素和关卡生成奖励
```

毒相关实现不要再用“中毒层数 > 0”推导毒附着。推荐敌人状态至少拆成：

```csharp
public int poisonStacks;
public ElementType elementAttachment;
public bool deepPoison;
```

毒牌处理规则：

```csharp
// 毒攻击：先按毒元素攻击尝试附着或反应，再造成攻击伤害。
DealAttackDamage(damage, hitCount, ElementType.Poison);

// 上毒：也要尝试附着或反应一次毒元素，然后增加中毒层数。
ResolveElementAttachment(ElementType.Poison);
ApplyPoisonStacks(amount);
```

元素反应表：

| 当前附着 | 新附着 | 反应 | 效果 |
| --- | --- | --- | --- |
| 火 | 水 | 蒸发 | 本次攻击伤害变为 1.5 倍。 |
| 水 | 火 | 蒸发 | 本次攻击伤害变为 1.5 倍。 |
| 火 | 毒 | 毒性加深 | 本次施加的中毒层数翻倍。 |
| 毒 | 火 | 毒性爆发 | 立即结算一次当前中毒层数伤害，然后中毒层数降低到 80%。 |
| 毒 | 水 | 深度中毒 | 敌人下一次攻击伤害降低。 |
| 水 | 毒 | 深度中毒 | 敌人下一次攻击伤害降低。 |

## 奖励稀有度曲线

| 阶段 | 普通 | 稀有 | 珍贵 |
| --- | ---: | ---: | ---: |
| 开局选牌 | 80% | 20% | 0% |
| 第 1-3 关战斗奖励 | 75% | 25% | 0% |
| 第 4-7 关战斗奖励 | 55% | 35% | 10% |
| 第 8-10 关战斗奖励 | 35% | 40% | 25% |

具体抽样权重由 RewardResolver 实现；本文件夹只提供可被抽样的候选池。
